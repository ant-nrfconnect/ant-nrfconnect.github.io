The Advanced Burst examples consists of an example application that can be used to test ANT burst transfers between a transmitter and receiver.

Overview

On dual core, this sample uses sysbuild to add a network core image to the build by defining the ‘’NRF_DEFAULT_ANT_ONLY’’ with a default value of ‘’y’’ in Kconfig.sysbuild.

Configuration

The sample configuration is split into the following two files:

  • generic configuration is available in the prj.conf file (single core, or dual core cpuapp)

  • configuration for the ant_rpc network core image is stored in the sysbuild/ant_rpc subdirectory (dual core cpunet)

Building and running

This sample can be found under ant/samples/ant_advanced_burst in the nRF Connect SDK folder structure.

Testing

After programming the sample to your development kit, you can test Burst Transfers using ANTware II:

  1. Compile and program the nRF5340 DK.

  2. Run ANTware II. Select your ANT PC dongle (for example, ANTUSB-m) from the available devices.

  3. Configure the device channel by loading the device profile configuration from the following file:

    ant\samples\ant_advanced_burst\ant_advanced_burst_test.xml

    Alternatively, you can configure the device channel manually:

    • Set the channel assignment to Slave.

    • Set the channel ID to “2, 2, 1”.

  4. From the “Adv” tab under the connected device window, select “Show Advanced Device Panel”. From the pop-up, select the “Adv. Burst” tab. Ensure that the “Enable” option is selected, “Pckts/AntMsg” is set to 3. Leave “Required” and “Optional” Fields alone. Press “Set Config” to apply the configuration.

  5. Click the Auto-Open button. Check the application core virtual COM port of the nRF5340 DK for logging output.
